# tasks/shared

Reglas comunes que toda Task debe cumplir y referenciar en lugar de duplicar (ver `tasks/README.md`, sección "Dependencias compartidas").

| Archivo | Propósito |
|---|---|
| `documentation-rules.md` | Principios y jerarquía de fuentes para producir documentación (Single Source of Truth, manejo de contradicciones, uso del Lenguaje Ubicuo). |
| `diagram-rules.md` | Reglas para la producción gráfica: selección de tipo de diagrama, composición, consistencia visual, restricciones técnicas. |
| `output-format.md` | Estructura obligatoria de salida de toda Task (Task Summary, Inputs, Analysis, Decisions, Artifact Specification, Generated Artifacts, Implementation, Validation, Issues, Completion Status). |
| `quality-checklist.md` | Checklist de validación que toda Task debe aplicar antes de marcarse como finalizada. |
